# CONTEXT.md — EAISI UWV

Machine-learning pipeline that forecasts Dutch sick-leave (*ziekteverzuim*) percentage **4 quarters ahead, per SBI sector**, from CBS open data. A mini-medallion data pipeline (raw → bronze → silver → gold) feeds a clean **7-step ML pipeline** with MLflow-backed evaluation and a per-sector `@prod` champion registry. Everything runs from `main.py`.

| Key | Value |
|-----|-------|
| Project Type | Time-series ML pipeline (per-sector forecasting) |
| Runtime | Python 3.10, managed by `uv` |
| Target | `Ziekteverzuimpercentage_1` (sick-leave %), 4Q-ahead, per SBI sector |
| Tracking + eval store | MLflow on one SQLite file — `data/4_eval/eval_data.db` |
| Entry point | `main.py` (CLI dispatch) |
| Agent context | [`AGENTS.md`](AGENTS.md) (local) · backlog [`ISSUES.md`](ISSUES.md) (local) |

## Glossary

| Term | Definition | Disambiguation |
|------|------------|----------------|
| **MASE** | THE comparison metric **and** champion gate: outer-fold MAE ÷ in-sample seasonal-naive m=4 MAE. Dimensionless; **< 1 beats the seasonal naive**. Computed in `ml_5`. | Not MAE/MAPE. The denominator is the *seasonal-naive*, **not** the `baseline` model. |
| **MAE (pp)** | Primary **stakeholder** headline — mean absolute error in **percentage points** (the target is itself a %). | "pp" = percentage points, not "%". Per-sector magnitude; not comparable across sectors (use MASE). |
| **MAPE** | Relative error `|y−ŷ|/|y|`, a **diagnostic** kept in the eval DB + `sector_quality.csv`. | A **fraction** (`0.05 == 5%`), not a percent. Demoted from the headline. |
| **`@prod` champion** | The lowest-MASE registered model for a sector (MLflow alias on `master_SickLeave_4Q_<sector>`). | One per sector; it **is** the cross-method winner of that sector. |
| **eval DB** | `data/4_eval/eval_data.db` — the **same** SQLite file is the MLflow backend store **and** holds the domain ORM tables (`model_evaluations`, `model_predictions`, `model_forecasts`, `sector_performance`, `model_tuning_results`). | Not a separate metrics DB; MLflow tracking + domain tables coexist. |
| **Gold store** | `data/3_gold/gold_data.db` — ML-ready feature store, panel of ~39 sectors × quarters, zero-null, float64. | Bronze/Silver are upstream; the ML pipeline only reads Gold. |
| **SBI sector** | CBS industry code, encoded as OHE column `BedrijfskenmerkenSBI2008_XXXXXX`. `T001081` = pre-computed **national total**. | `T001081` is the all-industry total row, not a sector; there is **no text SBI column** in Gold. |
| **Feature catalog** | `data/feature_selection/feature_catalog.json` — named feature groups (`all_survivors` = selected survivors). SSOT for groups. | Generated by `--select-features`; consumed by `model_configs.FEATURE_CATALOG`. |
| **Walk-forward / inner-outer folds** | Rolling-origin evaluation over 5 origins; **inner** folds pick the per-sector variant, **outer** folds score it honestly. | Headline metrics use **outer folds only** — never the inner folds. |
| **Family vs variant** | *Family* = `ModelConfiguration` name (`AutoETS_Stat`, `Ridge_Reduced`, …); *variant* = a specific run/preset of a family. | In the cross-method comparison the reducer families collapse into one `Pipeline` competitor. |
| **baseline** | `SectorQuarterRollingMean` — a competitor scored under the same MASE (`baseline_mase` reference). | NOT the MASE denominator (that is the seasonal-naive m=4 scaler in `ml_5`). |

## Architectural Constraints

Invariants that must not be violated.

- **Session protocol** — at session start read `README.md` → `CONTEXT.md` → `AGENTS.md`; at close-down update them with *durable* changes only (essentials, no bloat, prune stale). Full protocol: [`AGENTS.md`](AGENTS.md) §Session Protocol.
- **MASE is THE gate + comparison metric**; **MAE (pp)** is the stakeholder headline; **MAPE** is a diagnostic only. `ml_5` is the metric authority (sklearn); the shared primitives live in `m_evaluation` and `ml_5` draws from them.
- **`src/ml_engineering/` stays clean** — only numbered `ml_1…ml_7` + `ml_orchestrator` + `model_configs`. **No loose helper files**: fold shared logic into the owning step (e.g. `build_future_x` in `ml_5`) or `model_configs` (`QuarterlyPeriodForecaster`).
- **`src/utils/` helpers use the `m_` prefix.** Convention: utils = *generic/reusable*; case-specific reporting currently lives in `m_sector_quality`/`m_model_viz` (relocation into the domain is a **deferred** refactor — see `ISSUES.md`).
- **The eval DB *is* the MLflow backend** (`sqlite:///data/4_eval/eval_data.db`). The **MLflow registry (`@prod` alias) is the single source of truth**; `sector_performance` / `model_forecasts` are refresh-only projections (delete-then-insert).
- **No leakage** — feature selection drops the last 20 quarters; outer folds are never seen during variant selection; forecast-window X is production-honest (`build_future_x`, no covariate leakage).
- **Additive, no breaking changes** near the ship (week of 2026-06-19); never rename existing eval tables/columns.
- **Docs/narratives go to `.claude/`** (non-versioned), NOT the repo tree. `CONTEXT.md`, `README.md`, and `AGENTS.md` are the deliberate exceptions — the shared navigation/standing-instruction surface.
- **`uv` for everything** — `uv sync`, `uv run`. SQLite is the store; `mlruns/` holds model binaries only (gitignored).
- **Windows**: stop the MLflow UI (port 5000) before training / file ops — it locks the eval-DB SQLite file.
- **Imports** are absolute from `src.*`; every `src/` script guards with `if __name__ == "__main__":`.
